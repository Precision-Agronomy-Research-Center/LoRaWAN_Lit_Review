# Instructions for Contributing with GitHub

This document explains how to contribute to the project  
**“A Systematic Review of LoRaWAN for Agriculture Decision Support.”**

Contributions are welcome from researchers, engineers, practitioners, extension agents, and others with relevant experience.

---

## GitHub Vocabulary (Quick Reference)

- **Repository**  
  The collection of files and discussions for this review.

- **Issue**  
  A discussion thread used to ask questions, suggest changes, flag errors, or propose references.

- **Pull Request (PR)**  
  A request to merge your edits into the main document.

- **Manuscript Source**  
  The Markdown files that contain the review text.

---

## How to Contribute (Browser-Based)

This is the easiest method and works entirely in your web browser.

1. Navigate to the file you want to edit (usually in the manuscript directory).
2. Click the pencil icon (“Edit”) in the upper-right corner.
3. Make your changes using plain text (Markdown).
4. Scroll down and write a brief summary of your changes.
5. Select **“Create a new branch for this commit and start a pull request.”**
6. Submit the pull request.

A maintainer will review your contribution and follow up if revisions are needed.

---

## How to See and Revise Your Changes

Once you open a pull request:

- GitHub will display a comparison between your changes and the existing text.
- Review comments may be added by maintainers or other contributors.
- You can continue editing the same branch to address feedback.

---

## What to Contribute

Examples of valuable contributions include:

- Adding or correcting citations  
- Summarizing LoRaWAN deployments or case studies  
- Clarifying technical tradeoffs or assumptions  
- Highlighting limitations, failures, or gaps in the literature  
- Improving clarity, structure, or terminology  

Evidence-based contributions are strongly encouraged.

---

## Asking Questions or Getting Help

If you are unsure how to proceed, please open an **Issue** and describe:

- What you want to contribute
- What you are unsure about
- Any relevant references or experience

Questions are welcome and help improve the project for future contributors.

## Command Line Users (Optional)

If you are comfortable with Git:

1. Fork the repository.
2. Create a new branch for your changes.
3. Commit your edits (one sentence per line is preferred).
4. Open a pull request to the main repository.

Please keep commits focused and clearly described.

---

## Code of Conduct

All contributions are subject to the project’s  
[`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).

Professional, respectful, and constructive participation is expected.
